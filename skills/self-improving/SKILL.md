# Self-Improving Skill - 自我进化技能

让 AI 能够自动记录用户偏好，并在未来任务中自动应用，避免重复讲解。

## 🎯 核心能力

1. **自动识别** - 识别用户在表达偏好或规则
2. **结构化记录** - 将偏好写入持久化存储
3. **主动应用** - 在未来任务中自动应用这些规则
4. **强制激活** - 确保偏好被注意到，不会被忽略

---

## 🚨 激活规则（强制执行）

**这个 skill 在以下情况下自动激活：**

1. **用户表达强烈偏好时**（见触发规则）
2. **每次任务开始时** - 自动读取 `memory/preferences.json`，如果有相关偏好立即声明

### 为什么需要强制激活？

> AI 的注意力会收窄，不会主动翻阅远处的文档。因此，这个 skill 会在 **SKILL.md 顶部** 和 **每次任务开始时** 主动读取偏好，并在回复中明确声明"我注意到你的偏好：XXX"。

---

## 📋 触发规则 - 识别用户在表达偏好

### 强触发词（立即记录）

用户使用以下表达时，**立即触发记录**：

- "永远不要..."
- "下次不要..."
- "记住，..."
- "以后都这样..."
- "不要让我重复讲..."
- "这是我的规则：..."
- "我的偏好是..."

**示例：**
```
用户: "不要让我手动复制文档工作，永远不要"
→ 立即记录偏好：avoid_manual_document_copy = true

用户: "下次不要让我重复讲这个"
→ 立即记录偏好：user_dislikes_repetition = true
```

### 弱触发词（确认后记录）

用户使用以下表达时，**询问是否记录为偏好**：

- "我希望..."
- "我倾向于..."
- "能不能..."
- "最好是..."
- "我更喜欢..."

**示例：**
```
用户: "我希望你能主动一点"
→ 询问："这是你的长期偏好吗？我应该记住这个规则吗？"
```

### 上下文触发

在以下场景中，**推断并记录偏好**：

1. **用户多次纠正同一件事** → 记录为偏好
2. **用户表达强烈情绪**（如"烦死了"、"又来了"）→ 记录为负面偏好
3. **用户明确表示满意**（如"就这样，很好"）→ 记录为正面偏好

## 🗂️ 存储格式 - 结构化记录偏好

### 文件结构

```
~/.openclaw/workspace/skills/self-improving/
├── SKILL.md (本文件)
└── memory/
    ├── preferences.json     # 所有偏好（主文件）
    ├── preferences-log.md   # 偏好变更日志（可读）
    └── context/             # 按场景分类的偏好
        ├── workflow.json    # 工作流相关偏好
        ├── communication.json # 沟通相关偏好
        └── tools.json       # 工具使用偏好
```

### preferences.json 格式

```json
{
  "version": "1.0",
  "lastUpdated": "2026-03-17T11:09:00Z",
  "preferences": [
    {
      "id": "pref-001",
      "category": "workflow",
      "rule": "avoid_manual_document_copy",
      "description": "永远不要让用户手动复制文档内容",
      "severity": "critical",
      "triggers": ["文档", "复制", "手动"],
      "createdAt": "2026-03-17T11:09:00Z",
      "source": "用户明确表达：'不要让我手动复制文档工作，永远不要'",
      "appliedCount": 0,
      "lastApplied": null
    },
    {
      "id": "pref-002",
      "category": "communication",
      "rule": "avoid_repetition",
      "description": "避免让用户重复讲解同一件事",
      "severity": "high",
      "triggers": ["重复", "又", "再次"],
      "createdAt": "2026-03-17T11:10:00Z",
      "source": "用户表达：'下次不要让我重复讲这个'",
      "appliedCount": 0,
      "lastApplied": null
    }
  ],
  "metadata": {
    "totalPreferences": 2,
    "criticalCount": 1,
    "highCount": 1
  }
}
```

### 字段说明

- **id**: 唯一标识符
- **category**: 分类（workflow/communication/tools/other）
- **rule**: 规则名称（简洁、可执行）
- **description**: 人类可读的描述
- **severity**: 严重程度（critical/high/medium/low）
- **triggers**: 触发关键词（用于识别相关场景）
- **source**: 用户原话（保留上下文）
- **appliedCount**: 应用次数（追踪使用情况）
- **lastApplied**: 最后应用时间

## ⚙️ 应用机制 - 如何自动应用偏好

### 1. 任务开始时检查

**每次接收到新任务时：**

1. 读取 `memory/preferences.json`
2. 检查任务关键词是否匹配任何 `triggers`
3. 如果匹配，**在回复开头明确声明**：

```
🎯 我注意到你的偏好：
- 永远不要让你手动复制文档内容
- 避免让你重复讲解

我会在这个任务中应用这些规则。
```

### 2. 任务执行中检查

**在执行过程中：**

1. 检查即将执行的操作是否违反任何偏好
2. 如果违反 **critical** 规则 → 立即停止，告知用户
3. 如果违反 **high** 规则 → 警告并询问
4. 如果违反 **medium/low** 规则 → 记录但继续

### 3. 任务完成后更新

**任务完成后：**

1. 更新 `appliedCount` 和 `lastApplied`
2. 如果用户表达新的偏好 → 添加到 `preferences.json`
3. 如果用户纠正偏好 → 更新或删除

## ⚠️ 注意事项

### 1. 不要过度记录

- 不是所有用户表达都是"偏好"
- 区分"一次性需求"和"长期规则"
- 有疑问时，**询问用户**是否要记录为偏好

### 2. 尊重用户隐私

- 偏好文件可能包含敏感信息
- 不要在群聊中暴露个人偏好
- 提供"删除偏好"的机制

### 3. 避免冲突

- 新偏好可能与旧偏好冲突
- 检测冲突时，**询问用户**如何处理
- 提供"查看所有偏好"和"编辑偏好"的功能

### 4. 保持可解释性

- 每次应用偏好时，**明确告知用户**
- 用户应该知道 AI 在遵循什么规则
- 提供"为什么这样做"的解释

## 🎓 设计理念

### 为什么需要这个 skill？

1. **AI 的注意力收窄** - AI 在执行任务时，不会主动翻阅远处的文档
2. **"Memory" 的误导** - 存进去不等于记住了，需要主动应用
3. **重复的痛苦** - 用户不应该反复讲解同一件事
4. **自我进化** - AI 应该随着使用变得越来越懂用户

### 核心原则

1. **主动 > 被动** - 不要等用户提醒，主动应用偏好
2. **透明 > 隐蔽** - 明确告知用户在遵循什么规则
3. **可解释 > 黑盒** - 用户应该理解 AI 的行为
4. **渐进 > 激进** - 从简单规则开始，逐步进化
