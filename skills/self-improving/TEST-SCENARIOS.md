# Self-Improving Skill - 测试场景

这个文件提供一些测试场景，帮助你理解这个 skill 如何工作。

---

## 场景 1：识别并记录偏好

### 用户输入
```
"记住，我讨厌在群里发长篇大论，以后在群里回复要简短"
```

### 预期行为

1. **识别**：用户在表达沟通偏好
2. **记录**：
   ```json
   {
     "id": "pref-003",
     "category": "communication",
     "rule": "concise_in_groups",
     "description": "在群聊中保持回复简短",
     "severity": "high",
     "triggers": ["群", "简短", "长篇"],
     "source": "用户表达：'我讨厌在群里发长篇大论，以后在群里回复要简短'"
   }
   ```
3. **回复**：
   ```
   🎯 已记录你的偏好：
   - 规则：在群聊中保持回复简短
   - 严重程度：high
   - 分类：communication
   
   我会在群聊场景中应用这个规则。
   ```

## 场景 2：应用已记录的偏好

### 用户输入
```
"帮我在群里总结一下今天的项目进展"
```

### 预期行为

1. **检查偏好**：读取 `preferences.json`
2. **匹配触发词**："群" 匹配 `pref-003` 的 triggers
3. **声明偏好**：
   ```
   🎯 我注意到你的偏好：
   - 在群聊中保持回复简短
   - 永远不要让你手动复制文档内容
   
   我会在这个任务中应用这些规则。
   ```
4. **执行任务**：提供简短的项目进展总结，不超过 3-5 行

## 场景 3：避免违反偏好

### 用户输入
```
"帮我整理一下这个文档的内容，你告诉我需要复制哪些部分"
```

### 预期行为

1. **检查偏好**：读取 `preferences.json`
2. **检测冲突**：用户要求手动复制，违反 `pref-001` (critical)
3. **阻止并声明**：
   ```
   ⚠️ 我注意到你的偏好：
   - 永远不要让你手动复制文档内容（critical）
   
   这个要求违反了你的偏好。我会直接通过 API 获取文档内容，不需要你手动复制。
   
   请提供文档链接或文档 ID。
   ```

---

## 场景 4：更新偏好

### 用户输入
```
"其实手动复制也可以，只要不超过 100 字"
```

### 预期行为

1. **识别**：用户在更新现有偏好
2. **更新**：
   ```json
   {
     "id": "pref-001",
     "rule": "avoid_manual_document_copy",
     "description": "避免大量手动复制，少量（<100字）可以接受",
     "severity": "medium",  // 从 critical 降级为 medium
     "triggers": ["文档", "复制", "手动"],
     "updatedAt": "2026-03-17T12:00:00Z"
   }
   ```
3. **回复**：
   ```
   🎯 已更新你的偏好：
   - 旧规则：永远不要让你手动复制文档内容（critical）
   - 新规则：避免大量手动复制，少量（<100字）可以接受（medium）
   
   偏好已降级并更新描述。
   ```

## 场景 6：删除偏好

### 用户输入
```
"删除关于群聊简短的偏好"
```

### 预期行为

1. **识别**：用户要删除偏好
2. **确认**：
   ```
   确认删除以下偏好？
   - [communication - high] concise_in_groups
   - 在群聊中保持回复简短
   
   回复"确认"以删除，或"取消"以保留。
   ```
3. **执行**：用户确认后，从 `preferences.json` 中删除，并在 `preferences-log.md` 中记录
