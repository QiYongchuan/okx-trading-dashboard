# 飞书文档同步优化方案

## 一、背景与问题

### 当前实现方式
- 使用 `create-doc` MCP 工具
- 传入完整的 Markdown 内容字符串
- 飞书后端将 Markdown 转换为文档块（逐块插入）

### 存在的问题
1. **LLM Token 消耗大**：需要 LLM 输出完整的 Markdown 内容到参数中
2. **大文档不稳定**：长文本可能导致 API 超时或失败
3. **重复传输**：相同内容在 LLM → MCP → 飞书之间多次传输

---

## 二、飞书文件上传 API 分析

### 1. 媒体上传 API
**接口**: `POST /open-apis/drive/v1/medias/upload_all`

**用途**: 上传图片、文件等媒体到**已有文档**中

**关键参数**:
- `parent_type`: 上传点类型（如 `docx_image`, `docx_file`）
- `parent_node`: 文档块的 block_id
- `file`: 文件二进制内容

**限制**:
- 文件大小 ≤ 20MB
- 频率限制：5 QPS，每天 10,000 次

**适用场景**: 向已创建的文档中插入图片/附件，**不适用于创建新文档**

---

### 2. 文件导入 API（推荐方案）
**接口**: `POST /open-apis/drive/v1/import_tasks`

**用途**: 将本地文件（Word、Markdown、Excel 等）导入为飞书云文档

**支持的文件格式**:
- Markdown: `.md`, `.markdown`
- Word: `.docx`, `.doc`
- Excel: `.xlsx`, `.xls`
- 其他: `.txt`, `.csv` 等

**工作流程**:
```
1. 上传文件 → medias/upload_all (parent_type=ccm_import_open)
   ↓
2. 获得 file_token
   ↓
3. 创建导入任务 → import_tasks
   ↓
4. 轮询任务状态 → import_tasks/{ticket}
   ↓
5. 获取文档 token
```

**关键参数**:
```json
{
  "file_extension": "md",           // 文件扩展名
  "file_token": "boxcnrHpsg1...",   // 上传后获得的 token
  "type": "docx",                   // 目标类型：新版文档
  "file_name": "文档标题",           // 可选，文档名称
  "point": {
    "mount_type": 1,                // 固定值：挂载到云空间
    "mount_key": "fldbcO1UuPz8..."  // 文件夹 token，空则为根目录
  }
}
```

**限制**:
- 文件大小 ≤ 20MB
- file_token 有效期：5 分钟
- 异步任务，需要轮询结果

**优势**:
- ✅ **1 次 API 调用上传文件**（不需要逐块插入）
- ✅ **支持文件路径**（不需要 LLM 输出完整内容）
- ✅ **原生 Markdown 转换**（飞书后端处理）
- ✅ **支持大文件**（20MB 以内）

## 五、优化方案实现

### 方案 A：完全替换（推荐用于新文档）

**适用场景**: 从本地 .md 文件创建全新的飞书文档

**步骤**:
```javascript
// 1. 上传文件
const uploadRes = await fetch('https://open.feishu.cn/open-apis/drive/v1/medias/upload_all', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${accessToken}`,
    'Content-Type': 'multipart/form-data'
  },
  body: formData // 包含 file_name, parent_type=ccm_import_open, size, file
});

const { file_token } = uploadRes.data;

// 2. 创建导入任务
const importRes = await fetch('https://open.feishu.cn/open-apis/drive/v1/import_tasks', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${accessToken}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    file_extension: 'md',
    file_token: file_token,
    type: 'docx',
    file_name: '文档标题',
    point: {
      mount_type: 1,
      mount_key: 'folder_token_here' // 或空字符串表示根目录
    }
  })
});

const { ticket } = importRes.data;

// 3. 轮询任务状态
let result;
while (true) {
  const statusRes = await fetch(`https://open.feishu.cn/open-apis/drive/v1/import_tasks/${ticket}`, {
    headers: { 'Authorization': `Bearer ${accessToken}` }
  });
  
  if (statusRes.data.job_status === 0) { // 0 = 成功
    result = statusRes.data;
    break;
  }
  
  await sleep(1000); // 等待 1 秒后重试
}

// 4. 获取文档 token
const docToken = result.token;
```

### 方案 B：混合方案（推荐用于更新文档）

**适用场景**: 已有飞书文档，需要用 .md 文件内容更新

**思路**:
1. 先用方案 A 创建一个临时文档
2. 获取临时文档的内容（通过 fetch-doc）
3. 用 update-doc 更新目标文档

**代码示例**:
```javascript
// 1. 导入 .md 文件为临时文档
const tempDocToken = await importMarkdownFile('/path/to/file.md', '临时文档');

// 2. 获取临时文档内容
const tempDocContent = await fetchDoc(tempDocToken);

// 3. 更新目标文档
await updateDoc({
  doc_id: targetDocId,
  mode: 'overwrite',
  markdown: tempDocContent.markdown
});

// 4. 可选：删除临时文档
// await deleteDoc(tempDocToken);
```

### 方案 C：直接读取文件 + create-doc（折中方案）

**适用场景**: 不想申请新权限，但想减少 LLM token 消耗

**实现**:
```javascript
// 1. 直接读取本地 .md 文件
const fs = require('fs');
const markdownContent = fs.readFileSync('/path/to/file.md', 'utf-8');

// 2. 调用现有 create-doc（跳过 LLM 输出）
await createDoc({
  title: '文档标题',
  markdown: markdownContent,  // 直接传入文件内容
  folder_token: 'fldbcO1UuPz8...'
});
```

**优势**:
- ✅ 不需要新权限
- ✅ 减少 LLM token 消耗（LLM 不需要输出内容）
- ✅ 实现简单

**劣势**:
- ❌ 仍然需要传输完整 Markdown 字符串
- ❌ 大文档可能超时

### 第二阶段：完整优化（需要新权限）
**方案**: 方案 A - 文件导入 API

**前置条件**:
1. 申请 `docs:document:import` 权限
2. 等待权限审核通过（通常 1-3 天）

**实施步骤**:
1. 添加新工具 `feishu_import_markdown_file`
2. 实现完整的导入流程（上传→导入→轮询）
3. 处理异步任务状态查询
4. 添加错误处理和重试机制

**代码框架**:
```javascript
// 新工具定义
const ImportMarkdownSchema = Type.Object({
  file_path: Type.String({ description: '本地 .md 文件路径' }),
  title: Type.Optional(Type.String({ description: '文档标题（默认使用文件名）' }),
  folder_token: Type.Optional(Type.String({ description: '目标文件夹 token' }),
  task_id: Type.Optional(Type.String({ description: '异步任务 ID（用于查询状态）' })
});

export async function importMarkdownFile(params) {
  if (params.task_id) {
    // 查询任务状态
    return await pollImportTask(params.task_id);
  }
  
  // 1. 读取文件
  const fileBuffer = fs.readFileSync(params.file_path);
  const fileName = path.basename(params.file_path);
  
  // 2. 上传文件
  const fileToken = await uploadFile(fileBuffer, fileName);
  
  // 3. 创建导入任务
  const ticket = await createImportTask({
    file_extension: 'md',
    file_token: fileToken,
    type: 'docx',
    file_name: params.title || fileName,
    point: {
      mount_type: 1,
      mount_key: params.folder_token || ''
    }
  });
  
  // 4. 立即返回 task_id（异步处理）
  return {
    task_id: ticket,
    message: '导入任务已创建，请稍后查询状态'
  };
}
```

**收益**:
- ✅ **最优性能**：1 次文件上传 + 1 次导入
- ✅ **支持大文档**：≤ 20MB
- ✅ **完全异步**：不阻塞主流程
